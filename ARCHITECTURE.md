# Architecture: Next.js 15 Hybrid Modular SPA

## Stack

Next.js 15 (App Router), React 19, TypeScript strict, Tailwind v4, shadcn/ui, TanStack Query, Zustand, React Hook Form + Zod, TanStack Table, Sonner, Vitest + Testing Library + MSW, Storybook.

---

## Base Module Structure (12 files)

Every new module starts with EXACTLY this structure (12 files). Use `npx tsx scripts/make-module.ts --name={mod} --entity={Entity}`.

```
modules/{mod}/
├── api.ts                          # Query key factory + pure API functions
├── types.ts                        # Module-specific UI types + re-exports from generated API types
├── hooks.ts                        # ALL hooks (queries + mutations). Thin wrappers over api.ts
├── schemas.ts                      # Zod schemas for forms
├── components/
│   ├── {Entity}List.tsx            # Table with filters + pagination
│   ├── {Entity}List.stories.tsx    # Storybook stories
│   ├── {Entity}Form.tsx            # Create AND edit form (RHF + Zod)
│   ├── {Entity}Form.stories.tsx    # Storybook stories
│   └── columns.tsx                 # TanStack Table column defs
└── __tests__/
    └── {mod}.test.ts               # ALL tests in one file

app/(dashboard)/
├── {entities}/page.tsx             # List page
└── {entities}/new/page.tsx         # Create page
```

## File Responsibilities

### `api.ts` — Query Key Factory + Pure API Functions

No React, no hooks. Exports two things: the query key factory and pure async functions that call `httpClient`.

```typescript
import { httpClient } from "@/lib/http-client";
import type { Product, ProductListParams, CreateProductInput, UpdateProductInput, PaginatedResponse } from "./types";

// 1. Query key factory — single source of truth for cache keys
export const productKeys = {
  all: ["products"] as const,
  lists: () => [...productKeys.all, "list"] as const,
  list: (params: ProductListParams) => [...productKeys.lists(), params] as const,
  details: () => [...productKeys.all, "detail"] as const,
  detail: (id: string) => [...productKeys.details(), id] as const,
};

// 2. Pure API functions — no hooks, no React
export async function getProducts(params: ProductListParams): Promise<PaginatedResponse<Product>> {
  return httpClient.get("/api/products", { params });
}

export async function getProduct(id: string): Promise<Product> {
  return httpClient.get(`/api/products/${id}`);
}

export async function createProduct(data: CreateProductInput): Promise<Product> {
  return httpClient.post("/api/products", data);
}

export async function updateProduct(id: string, data: UpdateProductInput): Promise<Product> {
  return httpClient.put(`/api/products/${id}`, data);
}
```

### `types.ts` — Module Types

Two purposes: re-export generated API types that this module uses, and define module-specific UI types (filter state, component props, derived types).

```typescript
// Re-export API types this module needs (generated by npm run generate:types)
export type {
  Product,
  CreateProductInput,
  UpdateProductInput,
} from "@/types/api";

// Module-specific UI types (not in the API)
export interface ProductListParams {
  search?: string;
  status?: string;
  page?: number;
  per_page?: number;
}

export interface ProductFormProps {
  defaultValues?: Partial<CreateProductInput>;
  productId?: string;
  mode: "create" | "edit";
}
```

**Rule:** `types/api.ts` is auto-generated and read-only. `modules/{mod}/types.ts` is the module's curated surface — other files in the module import from here, never directly from `types/api.ts`.

### `hooks.ts` — Thin React Wrappers

Connects `api.ts` to React via TanStack Query. No business logic, no data transformation — just wiring.

```typescript
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import { productKeys, getProducts, createProduct } from "./api";
import type { ProductListParams } from "./types";

export function useProducts(params: ProductListParams) {
  return useQuery({
    queryKey: productKeys.list(params),
    queryFn: () => getProducts(params),
  });
}

export function useCreateProduct() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: createProduct,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: productKeys.lists() });
    },
  });
}

// useProduct(id) and useUpdateProduct(id) follow the same patterns
```

### `schemas.ts` — Zod Schemas for Forms

One schema per entity. Used by both create and edit forms.

```typescript
import { z } from "zod";

export const productSchema = z.object({
  name: z.string().min(1, "Name is required"),
  sku: z.string().min(1, "SKU is required"),
  price: z.coerce.number().min(0, "Price must be positive"),
  status: z.enum(["active", "inactive"]).default("active"),
});

export type ProductFormValues = z.infer<typeof productSchema>;
```

### `{Entity}Form.tsx` — Create AND Edit Form

One component for both modes. Receives `defaultValues` for edit mode.

```tsx
"use client";

import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { useRouter } from "next/navigation";
import { toast } from "sonner";
import { Form, FormControl, FormField, FormItem, FormLabel, FormMessage } from "@/components/ui/form";
import { Input } from "@/components/ui/input";
import { Button } from "@/components/ui/button";
import { mapApiErrors } from "@/lib/errors";
import { labels } from "@/lib/labels";
import { productSchema, type ProductFormValues } from "../schemas";
import { useCreateProduct, useUpdateProduct } from "../hooks";
import type { ProductFormProps } from "../types";

export function ProductForm({ defaultValues, productId, mode }: ProductFormProps) {
  const router = useRouter();
  const createMutation = useCreateProduct();
  const updateMutation = useUpdateProduct(productId ?? "");

  const form = useForm<ProductFormValues>({
    resolver: zodResolver(productSchema),
    defaultValues: defaultValues ?? { name: "", sku: "", price: 0, status: "active" },
  });

  async function onSubmit(values: ProductFormValues) {
    try {
      if (mode === "edit" && productId) {
        await updateMutation.mutateAsync(values);
        toast.success(labels.products.updated);
      } else {
        await createMutation.mutateAsync(values);
        toast.success(labels.products.created);
      }
      router.push("/products");
    } catch (error) {
      mapApiErrors(error, form.setError);
    }
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-4">
        <FormField
          control={form.control}
          name="name"
          render={({ field }) => (
            <FormItem>
              <FormLabel>{labels.products.fields.name}</FormLabel>
              <FormControl><Input {...field} /></FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        {/* Repeat FormField pattern for each field */}
        <Button type="submit" disabled={createMutation.isPending || updateMutation.isPending}>
          {mode === "edit" ? labels.common.save : labels.products.create}
        </Button>
      </form>
    </Form>
  );
}
```

### `{Entity}List.tsx` — Table with Filters and Pagination

Uses TanStack Table via the shared `DataTable` component. Filters and pagination sync with URL search params.

```tsx
"use client";

import { useSearchParams, useRouter, usePathname } from "next/navigation";
import { DataTable } from "@/components/ui/data-table";
import { Input } from "@/components/ui/input";
import { columns } from "./columns";
import { useProducts } from "../hooks";
import { labels } from "@/lib/labels";
import { LoadingState, EmptyState, ErrorState } from "@/components/ui/states";
import type { ProductListParams } from "../types";

export function ProductList() {
  const searchParams = useSearchParams();
  const router = useRouter();
  const pathname = usePathname();

  const params: ProductListParams = {
    search: searchParams.get("search") ?? undefined,
    status: searchParams.get("status") ?? undefined,
    page: Number(searchParams.get("page") ?? 1),
    per_page: 15,
  };

  const { data, isLoading, isError, error } = useProducts(params);

  function updateParams(updates: Partial<ProductListParams>) {
    const next = new URLSearchParams(searchParams.toString());
    Object.entries(updates).forEach(([key, value]) => {
      if (value) next.set(key, String(value));
      else next.delete(key);
    });
    next.set("page", "1");
    router.push(`${pathname}?${next.toString()}`);
  }

  if (isLoading) return <LoadingState />;
  if (isError) return <ErrorState error={error} />;
  if (!data?.data.length) return <EmptyState message={labels.products.empty} />;

  return (
    <div className="space-y-4">
      <Input
        placeholder={labels.products.searchPlaceholder}
        defaultValue={params.search}
        onChange={(e) => updateParams({ search: e.target.value || undefined })}
      />
      <DataTable columns={columns} data={data.data}
        pageCount={Math.ceil(data.meta.total / data.meta.per_page)}
        currentPage={data.meta.current_page}
        onPageChange={(page) => updateParams({ page })}
      />
    </div>
  );
}
```

### `columns.tsx` — TanStack Table Column Definitions

Uses `ColumnDef<T>[]` array. Actions column uses a dropdown menu with permission checks.

```tsx
import { ColumnDef } from "@tanstack/react-table";
import { Badge } from "@/components/ui/badge";
import { ActionsDropdown } from "@/components/ui/actions-dropdown";
import { labels } from "@/lib/labels";
import type { Product } from "../types";

export const columns: ColumnDef<Product>[] = [
  { accessorKey: "name", header: labels.products.fields.name },
  { accessorKey: "sku", header: labels.products.fields.sku },
  {
    accessorKey: "status",
    header: labels.products.fields.status,
    cell: ({ row }) => (
      <Badge variant={row.original.status === "active" ? "default" : "secondary"}>
        {labels.products.statuses[row.original.status]}
      </Badge>
    ),
  },
  {
    id: "actions",
    cell: ({ row }) => (
      <ActionsDropdown
        viewHref={`/products/${row.original.id}`}
        editHref={`/products/${row.original.id}/edit`}
        viewPermission="catalog.view"
        editPermission="catalog.update"
      />
    ),
  },
];
```

## Shared Infrastructure Contracts

These utilities exist in `lib/` and are used by all modules. Documented here so modules use them correctly.

### `lib/http-client.ts`

```typescript
// Public interface — do not import internals
httpClient.get<T>(url: string, options?: { params?: Record<string, unknown> }): Promise<T>
httpClient.post<T>(url: string, data?: unknown): Promise<T>
httpClient.put<T>(url: string, data?: unknown): Promise<T>
httpClient.patch<T>(url: string, data?: unknown): Promise<T>
httpClient.delete(url: string): Promise<void>
```

- Handles CSRF cookie automatically (419 → retry with fresh cookie).
- Sends `credentials: "include"` and `X-Requested-With: XMLHttpRequest`.
- Throws typed errors: `ApiError` with `status`, `message`, `errors` (field-level), `code` (business error).

### `lib/errors.ts`

```typescript
mapApiErrors(error: unknown, setError: UseFormSetError<any>): void
```

- **422 with field errors:** Calls `setError(field, { message })` per field.
- **422 with business error code:** Shows `toast.error(message)`.
- **401:** Redirects to login (handled by AuthGuard, but defensive here too).
- **Other:** Shows `toast.error("Unexpected error")`.

### `lib/labels.ts`

```typescript
// Structure: nested object by module
```

- **Rule:** Never hardcode user-facing text. Import from `labels`.
- **New modules** add their section to this file.

### `config/env.ts`

```typescript
import { env } from "@/config/env";
env.NEXT_PUBLIC_API_URL // string — validated at build time via Zod
```

- **Rule:** Never use `process.env` directly. Always `import { env } from "@/config/env"`.

### `lib/permissions.ts`

```typescript
can(permission: string): boolean
canAny(permissions: string[]): boolean
<Authorized permission="mod.action">{children}</Authorized>
<Authorized permissions={["mod.view", "mod.create"]} any>{children}</Authorized>
```

- `can()` reads from the user's permission list (loaded by `useUser()`).
- `<Authorized>` renders children only if the user has the permission. Renders nothing otherwise.

### `components/ui/states.tsx`

Standard UI states used across all modules:

```typescript
<LoadingState />                              // Spinner centered in container
<EmptyState message="No data available" />     // Icon + message + optional action button
<ErrorState error={error} />                  // Error message with retry button
<NotFoundState />                             // 404 with back navigation
```

- **Rule:** Every list must handle loading, empty, and error states.
- **Rule:** Every detail page must handle loading, not found, and error states.

## Conventions

| Area | Rule |
|------|------|
| Files | kebab-case. Components: PascalCase. |
| Imports | `@/` alias. No cross-module imports (except `types.ts`). |
| State | Server → TanStack Query. Global UI → Zustand. Local → useState. |
| API | Always via `api.ts` → `httpClient`. Never direct `fetch()`. |
| Forms | Zod schema → RHF `useForm` → shadcn `FormField` → `mutateAsync` + `mapApiErrors`. One component for create AND edit (mode prop). |
| Tests | Vitest + Testing Library + MSW. Never `vi.mock`. One file per module. |
| Stories | Every component gets `.stories.tsx`. Self-contained, no backend. |
| Pages | `app/` has no business logic — only routing. |
| No barrels | No `index.ts` re-exports. |
| URL state | Filters and pagination sync with URL search params. Use `useSearchParams()` + `router.push()`. |
| UI states | Every data-dependent view handles: loading (`LoadingState`), empty (`EmptyState`), error (`ErrorState`). |
| Labels | All user-facing text from `lib/labels.ts`. Never hardcoded. |
| Env | `import { env } from "@/config/env"`. Never `process.env`. |
| IDs | Always `string`. Never `number`. |
| Timestamps | Always `string`. |
| TypeScript | Strict mode. Never `any`. |

### Naming Conventions

| Type | Pattern | Example |
|------|---------|---------|
| API functions | `get{Entity}s`, `create{Entity}` | `getProducts`, `createProduct` |
| Query keys | `{entity}Keys.all`, `.list()`, `.detail(id)` | `productKeys.list(params)` |
| Hooks | `use{Entity}s`, `use{Action}{Entity}` | `useProducts`, `useCreateProduct` |
| Components | PascalCase | `ProductList`, `ProductForm` |
| Test files | `{mod}.test.ts` | `products.test.ts` |
| Story files | `{Component}.stories.tsx` | `ProductForm.stories.tsx` |
| Schemas | `{entity}Schema` | `productSchema` |
| Form values | `{Entity}FormValues` | `ProductFormValues` |

## Extension Points

| Trigger | Add |
|---------|-----|
| >10 hooks in one file | Split `hooks.ts` → `hooks/use-{name}.ts` |
| Detail page needed | `app/(dashboard)/{entities}/[id]/page.tsx` |
| Edit page needed | `app/(dashboard)/{entities}/[id]/edit/page.tsx` + reuse `{Entity}Form` with mode="edit" |
| State shared across routes | Zustand store in `modules/{mod}/store.ts` |
| Complex table filters | `components/{entity}-filters.tsx` |
| Real-time updates | `hooks/use-{mod}-channel.ts` (Laravel Echo) |

## Auth

- CSRF cookie before login: `GET /sanctum/csrf-cookie`
- `useUser()` for auth state. `AuthGuard` in dashboard layout.
- 419 → automatic CSRF retry in `httpClient`.
- Logout: `queryClient.clear()`.
